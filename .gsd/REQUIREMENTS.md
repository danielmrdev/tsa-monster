### R001 — Site built with Astro 6, output 100% static, i18n configured with `prefixDefaultLocale: true`. All pages live under `/en/`. Root `/` redirects to the correct locale via client-side language detection, fallback to `/en/`.
- Class: core-capability
- Status: validated
- Description: Site built with Astro 6, output 100% static, i18n configured with `prefixDefaultLocale: true`. All pages live under `/en/`. Root `/` redirects to the correct locale via client-side language detection, fallback to `/en/`.
- Why it matters: Foundation for every other requirement. Without this, nothing else works. The i18n structure must be right from the start — retrofitting it is expensive.
- Source: user
- Primary owning slice: M001/S01
- Supporting slices: none
- Validation: S01: pnpm build exits 0 with static output, 2 pages built under /en/. i18n prefixDefaultLocale:true confirmed in astro.config.mjs. pnpm astro check 0 errors (14 files). Redirect script at / confirmed by dev server curl.
- Notes: Astro 6 native i18n — no third-party i18n library needed.

### R006 — Root `/` redirects to the user's browser language if supported, fallback to `/en/`. When new locales are added, the redirect script picks them up automatically (driven by the `locales` array in config).
- Class: core-capability
- Status: validated
- Description: Root `/` redirects to the user's browser language if supported, fallback to `/en/`. When new locales are added, the redirect script picks them up automatically (driven by the `locales` array in config).
- Why it matters: UX baseline for a multi-locale site. Users arriving at the root get the right language without manual selection.
- Source: user
- Primary owning slice: M001/S01
- Supporting slices: none
- Validation: S01: src/pages/index.astro uses navigator.language + navigator.languages + isSupportedLocale() from src/i18n/config.ts. Redirect to /en/ confirmed in page source. LOCALES-driven — new locale in config automatically extends redirect logic.
- Notes: Client-side only (static site). Uses `navigator.language` + `navigator.languages`.
